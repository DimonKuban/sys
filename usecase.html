BookingMedicalConsultationAppointment
Use Case

Use case model 1

Business Use Case Name: СreateMedicalConsultationAppointment
Description: This use case enables Client or Receptionist to create an appointment
Actors: Client, Receptionist
Trigger: When client or Receptionist request consultation
Results: Medical consultation is scheduled

Use case model 2

Business Use Case Name: CancelMedicalConsultationAppointment
Description: This use case enables Client or Receptionist to cancel an appointment
Actors: Client, Receptionist
Trigger: When client or receptionist request to cancel consultation
Results: Medical consultation is canceled

Use case model 3

Business Use Case Name: UpdateMedicalConsultationAppointmentInformation
Description: This use case enables client and receptionist to update an appointment
Actors: Client, Receptionist
Trigger: When Medical Consultation schedule is created or canceled
Results: Medical consultation is rescheduled

Use case model 4

Business Use Case Name: RecordMedicalConsultationAppointmentInformation
Description: This use case enables Medical Staff and IT Staff to record an appointment
Actors: IT Staff, Medical Staff
Trigger: When booking system is activated
Results: Medical Consultation appointment information is recorded into the system

Use case model 5

Business Use Case Name: BackUpMedicalConsultationAppointmentInformation
Description: This use case enables to Keep information safe and back up to avoid data loses
Actors: IT Staff
Trigger: When Appointment information needs to be backed up.
Results: The information will be backed up

Use case model 6

Business Use Case Name: SendMedicalConsultationAppointmentInformation
Description: This use case enables the receptionist to inform client about their appointment information
Actors: Client, Receptionist
Trigger: When Receptionist confirm or decline appointment and client requests appointment information
Results: Client will get informed about application status of consultation

Use case model 7

Business Use Case Name: RecieveMedicalConsultationAppointmentInformation
Description: This use case enable client to recieve information about appointment
Actors: Client, Receptionist
Trigger: When appointment information sends to client
Results: Client recieves the information

Use case model 8

Business Use Case Name: PayMedicalConsultationFee
Description: This use case enables the client to pay for Medical consultation.
Actors: Client, Receptionist
Trigger: When client pays fee
Results: The consultation fee is paid

Use case model 9

Business Use Case Name: RefundMedicalConsultationFee
Description: This use case enables client to get refund after cancelling the appointment
Actors: Client, Receptionist
Trigger: When appointment was cancelled and it’s required to send money back to the client
Results: Client gets full or partial refund

Use case model 10

Business Use Case Name: RequestData
Description: This use case enables IT department to retrieve information from database and send it to the department who request information
Actors: IT Staff, Medical Staff
Trigger: When information is required (eg client phone number, appointment date, time etc.)
Results: Required information sent or request declined

Use case model 11

Business Use Case Name: CreateUserAccount
Description: This use case enables to IT staff to create user account in the system and adds benefits for signed client
Actors: Client, IT Staff
Trigger: When a new client wants to open an account with the business
Results: Client account is created

Use case model 12

Business Use Case Name: DiscountFees
Description: This use case enables to receptionist to discount fees for signed members
Actors: IT Staff, Client, Receptionist
Trigger: When a new client opens an account with the business
Results: The payment is deducted

Use case model 13

Business Use Case Name: ConfirmAppointment
Description: This use case enables receptionist to confirm the appointment based on client's information
Actors: IT Staff, Client, Receptionist
Trigger: When client create the appointment
Results: The appointment is confirmed

Use case model 14

Business Use Case Name: DeclineAppointment
Description: This use case enables receptionist to decline the appointment based on client's information (ex)Minor
Actors: IT Staff, Client, Receptionist
Trigger: When client create the appointment
Results: The appointment is declined
