UC2-CONFIRMATION

Business rule(s):

Rule 1: Appointment should exist.

Rule 2: There should be 2 days between now and appointment day.

Rule 3: The system should contain client information.

Description:

Client receive details about confirmation of an appointment.

Scenario 1:  

Confirm appointment.

Preconditions:

Client wants to confirm booked appointment.

Step# Actor (Client) Actor (Client) Data Used
1 Select a language

Display languages available options.

Default - English
LanguageList
Loop      
2  Select general appointment information  

Display information about upcoming appointment, including location, date, time, estimated price, doctor name
 AppointmentInfo
3  Select appointment details  Display information about items that patient should have when coming to appointment  RequiredItemsList
4  Select confirm booking acknowledgment  Display company polic, including cancelling and penalties for not showing up  PolicyInfo
5  Select confirm  Display confirmation number along with short booking information  ConfirmationDetails
End Loop  Select exit  Display main menu  

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 Successful Post-Conditions: client has got confirmation number and details about upcoming appointment, accept company policy and got important information about items that he/she should have for the appointment.
